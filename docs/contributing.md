# Contributing to HaltRescue Platform

Thank you for your interest in contributing to HaltRescue Platform! This project is open source and welcomes contributions from developers, designers, and animal welfare advocates.

## How to Contribute

Contributions can take many forms: code, documentation, bug reports, feature requests, and testing. All are valuable.

**Reporting Bugs:** If you find a bug, please open an issue on GitHub with a clear description of the problem, steps to reproduce it, and the expected vs. actual behavior.

**Suggesting Features:** If you have an idea for a new feature, please open an issue to discuss it before starting development. This ensures the feature aligns with the project's goals and avoids duplicated effort.

**Submitting Code:** To submit a code contribution, please fork the repository, create a new branch from `main` for your changes, and submit a pull request with a clear description of what you have changed and why.

## Development Setup

Please refer to the [Backend Setup Guide](./backend-setup.md) and [Frontend Setup Guide](./frontend-setup.md) for instructions on setting up a local development environment.

## Code Standards

All PHP code should adhere to the [Drupal Coding Standards](https://www.drupal.org/docs/develop/standards). All JavaScript/TypeScript code should be formatted with [Prettier](https://prettier.io/) and linted with [ESLint](https://eslint.org/).

## Code of Conduct

This project is committed to providing a welcoming and inclusive environment for all contributors. Please be respectful and constructive in all interactions.
